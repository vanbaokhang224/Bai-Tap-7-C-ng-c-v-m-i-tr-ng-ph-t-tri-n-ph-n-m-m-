Khái niệm CI/CD

CI - Continuous Integration

Tự động build và test code mỗi khi có thay đổi. Giúp phát hiện lỗi sớm.

  • Automatically build on every push
  • Run automated tests
  • Code quality checks

CD - Continuous Deployment

Tự động deploy phiên bản mới lên môi trường staging/production sau khi CI pass.

  • Automatic deployment to hosting
  • Zero-downtime releases
  • Rollback capabilities

Các bước thực hiện

  1. Commit & Push

    Developer push code lên GitLab repository.

  2. Pipeline Triggered

    GitLab CI tự động kích hoạt pipeline dựa trên .gitlab-ci.yml.

  3. Run Jobs

    Runner thực thi các jobs: build, test, deploy theo thứ tự stages.

  4. Deploy to Pages

    Nếu tất cả jobs thành công, artifacts được deploy lên GitLab Pages.

  5. Live Site

    Website được cập nhật tự động tại URL: https://[username].gitlab.io/[project-name]/

Truy cập CI/CD Pipeline

Xem Pipeline: Repository → CI/CD → Pipelines
Xem Jobs: Repository → CI/CD → Jobs
Xem Pipeline Editor: Repository → CI/CD → Editor

GitLab CI/CD Concepts

.gitlab-ci.yml File cấu hình pipeline bằng YAML, đặt ở root của repository.
Pipeline Tập hợp các jobs được chạy theo thứ tự stages.
Stage Nhóm các jobs chạy cùng lúc (VD: build, test, deploy).
Job Đơn vị nhỏ nhất trong pipeline, chứa các lệnh script thực thi.
Runner Server thực thi các jobs. GitLab.com cung cấp shared runners miễn phí.
Artifacts Các file được tạo ra bởi jobs, có thể chia sẻ giữa các stages/jobs.
